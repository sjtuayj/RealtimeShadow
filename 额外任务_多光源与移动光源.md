# 额外任务：多光源 + 移动光源

## 一、原理

### 1.1 多光源渲染

单光源只考虑一个方向的入射光。真实场景中通常有多个光源，每个光源对场景的贡献应该**线性累加**：

```
总radiance = Σᵢ (ambient/N + visibilityᵢ × (diffuseᵢ + specularᵢ))
```

其中 ambient 只需加一次（在第一个光源 pass 中处理），后续光源只累加直接光照。

### 1.2 实现方式：多 Pass + Additive Blending

在同一个帧缓冲上做两次绘制，第二次用**加法混合**：

```
Pass 1（光源0）: normal rendering
  输出 = ambient + visibility₀ × (diffuse₀ + specular₀)

Pass 2（光源1）: additive blending (gl.blendFunc(ONE, ONE))
  输出 = 已有颜色 + visibility₁ × (diffuse₁ + specular₁)
```

**为什么 ambient 只在 Pass 1 加**：如果每个 pass 都加 ambient，多光源会叠加出过亮的环境光。`uApplyAmbient` uniform 控制是否加 ambient（1.0 vs 0.0）。

### 1.3 每个光源独立 Shadow Map

两个光源各有一个 8192×8192 的 Shadow Map FBO。每帧：

```
for each light:
  Shadow Pass → light.fbo  （写入该光源的 shadow map）
  Camera Pass → 更新该光源的 uniforms（uLightMVP, uShadowMap, uLightIntensity）
```

### 1.4 移动光源

光源位置在主循环中每帧更新。由于 `lightMVP` 依赖光源位置，需要在每次绘制前**动态重新计算**：

```js
let mvp = light.CalcLightMVP(translate, scale);
material.uniforms.uLightMVP.value = mvp;
```

### 1.5 数据流

```
每帧 render():
  gl.clear()

  Light 0 (固定暖光):
    ├→ Shadow Pass: bind light0.fbo, clear, 绘制所有 shadowMeshes[0]
    ├→ Camera Pass: normal render (uApplyAmbient=1.0)
    │     └→ ambient + vis0×(diff0+spec0)

  Light 1 (轨道冷光):
    ├→ Shadow Pass: bind light1.fbo, clear, 绘制所有 shadowMeshes[1]
    ├→ Camera Pass: additive blend (uApplyAmbient=0.0)
    │     └→ 叠加 vis1×(diff1+spec1) 到已有颜色上

  Debug Quad (if enabled)
```

## 二、代码修改

共修改 **5 个文件**。

### 修改1：[phongFragment.glsl](src/shaders/phongShader/phongFragment.glsl) — ambient 开关

新增 uniform：

```glsl
uniform float uApplyAmbient;  // 1.0 = 加ambient, 0.0 = 只加直接光照
```

修改 `blinnPhong()`：

```glsl
// 之前: vec3 radiance = ambient + visibility * (diffuse + specular);
vec3 radiance = uApplyAmbient * ambient + visibility * (diffuse + specular);
```

→ 第一个光源 pass 设 `1.0`，后续 pass 设 `0.0`，避免重复加 ambient。

### 修改2：[PhongMaterial.js](src/materials/PhongMaterial.js) — 新增 uniform + 光源引用

```js
// 新增 uniform
'uApplyAmbient': { type: '1f', value: 1.0 },

// 存储光源引用，供 WebGLRenderer 动态更新
this._light = light;
```

### 修改3：[loadOBJ.js](src/loads/loadOBJ.js) — 每个光源创建独立 ShadowMaterial

**原代码**：只从 `renderer.lights[0]` 取一个光源创建 ShadowMaterial。

**新代码**：遍历所有光源，为每个启用了 Shadow Map 的光源创建独立的 ShadowMaterial：

```js
for (let li = 0; li < renderer.lights.length; li++) {
    let light = renderer.lights[li].entity;
    if (light.hasShadowMap) {
        let sm = buildShadowMaterial(light, Translation, Scale, ...);
        sm.then((data) => {
            renderer.addShadowMeshRender(li, new MeshRender(renderer.gl, mesh, data));
        });
    }
}
```

→ `shadowMeshes` 改为二维数组：`shadowMeshes[lightIndex][meshIndex]`。

### 修改4：[WebGLRenderer.js](src/renderers/WebGLRenderer.js) — 多光源 + additive blending + 动态 MVP

**核心改动**：

1. **`addShadowMeshRender(lightIndex, mesh)`** — 按光源索引组织 shadow mesh

2. **`_updateLightUniforms(mr, light, isFirstLight)`** — 每帧更新当前光源相关 uniforms：
   ```js
   // 动态计算 lightMVP（支持移动光源）
   let mvp = light.CalcLightMVP(translate, scale);
   mr.material.uniforms.uLightMVP.value = mvp;
   mr.material.uniforms.uShadowMap.value = light.fbo;
   mr.material.uniforms.uLightIntensity.value = light.mat.GetIntensity();
   mr.material.uniforms.uApplyAmbient.value = isFirstLight ? 1.0 : 0.0;
   ```

3. **`render()` 循环**：
   ```js
   for (let l = 0; l < this.lights.length; l++) {
       // Shadow pass（只绘制该光源的 shadowMeshes）
       gl.bindFramebuffer(gl.FRAMEBUFFER, light.fbo);
       gl.clear(...);
       for (let sm of this.shadowMeshes[l]) { sm.draw(camera); }

       // Camera pass（第二个光源起用 additive blending）
       if (!isFirstLight) {
           gl.enable(gl.BLEND);
           gl.blendFunc(gl.ONE, gl.ONE);
       }
       for (let mr of this.meshes) {
           this._updateLightUniforms(mr, light, isFirstLight);
           mr.draw(camera);
       }
       if (!isFirstLight) gl.disable(gl.BLEND);
       isFirstLight = false;
   }
   ```

### 修改5：[engine.js](src/engine.js) — 双光源 + 移动动画

```js
// Light 0: 固定暖色光（右上方）
const light0 = new DirectionalLight(3000, [1.0, 0.95, 0.9], [0, 80, 80], ...);

// Light 1: 轨道冷色光（初始在右侧，每帧旋转）
const light1 = new DirectionalLight(2500, [0.7, 0.8, 1.0], [80, 70, 0], ...);

// mainLoop 中更新轨道
light1.lightPos[0] = Math.cos(now * 0.0005) * 100;  // X 轴轨道
light1.lightPos[2] = Math.sin(now * 0.0005) * 100;  // Z 轴轨道
```

GUI 新增 "Animate Light 1" 复选框（`window.animateLight`），可开关轨道动画。

## 三、当前参数

| 参数 | 值 | 说明 |
|------|-----|------|
| Light 0 intensity | 3000 | 固定暖光 |
| Light 0 color | (1.0, 0.95, 0.9) | 暖色调 |
| Light 0 position | (0, 80, 80) | 右上方固定 |
| Light 1 intensity | 2500 | 移动冷光 |
| Light 1 color | (0.7, 0.8, 1.0) | 冷色调 |
| Light 1 orbit radius | 100 | XZ 平面绕圈半径 |
| Light 1 orbit speed | 0.0005 rad/ms | ≈ 0.5 rad/s |
| SHADOW_BIAS | 0.012 | 双光源需要更大 bias 防止叠加自阴影 |
| 每个光源 FBO | 8192×8192 | 独立 Shadow Map |

## 四、调试方向

| 现象 | 可能原因 |
|------|---------|
| 地面有多层暗影 | bias 不够大，双光源叠加 shadow acne → 增大 `SHADOW_BIAS` |
| 阴影与物体脱离（peter-panning） | bias 太大 → 减小 `SHADOW_BIAS` |
| 第二个光源没有阴影 | `light.hasShadowMap` 是否为 true |
| 固定光源的阴影不动、移动光源的阴影跟随 | 正常！每个光源独立 Shadow Map |
| 动画卡顿 | Shadow Map 分辨率 8192×2 开销大；降低分辨率或采样数 |
| 画面过亮 | additive blending 导致双光源叠加过亮 → 降低光源 intensity |
