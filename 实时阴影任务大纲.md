## PDF 作业目标

按照说明文档，作业要求实现 WebGL 实时阴影：

1. 硬阴影 Shadow Map
   - 使用 Two Pass Shadow Map。
   - 第一遍从光源视角渲染场景，生成 shadow map。
   - 第二遍从相机视角正常渲染，用 shadow map 判断当前片元可见性。

2. PCF
   - 在硬阴影基础上做 Percentage Closer Filter。
   - 使用圆盘采样核对 shadow map 多次采样。
   - 平均多次 shadow test 的结果，减少锯齿并柔化边缘。

3. PCSS
   - 实现 Percentage Closer Soft Shadow。
   - 先搜索 blocker 平均深度。
   - 根据 receiver 和 blocker 的深度关系估计半影大小。
   - 用动态 filter size 调用 PCF。
   - 得到接触处硬、远离遮挡物处更软的阴影。

4. 截图和提交
   - 新建 `images` 文件夹保存运行截图。
   - 截图分别展示硬阴影、PCF、PCSS。
   - 命名格式类似 `SM_xxx.png`、`PCF_xxx.png`、`PCSS_xxx.png`。
   - 提交时删除 `/lib` 和 `/assets` 文件夹。

## 任务 1：Shadow Map 硬阴影具体实现打算

任务 1 主要涉及三个文件：

- `src/lights/DirectionalLight.js`
- `src/shaders/phongShader/phongFragment.glsl`
- 必要时检查 `src/renderers/WebGLRenderer.js`

### 1. 实现 `CalcLightMVP(translate, scale)`

当前 `DirectionalLight.CalcLightMVP()` 中已经创建了：

```js
let lightMVP = mat4.create();
let modelMatrix = mat4.create();
let viewMatrix = mat4.create();
let projectionMatrix = mat4.create();
```

计划补充：

```js
mat4.identity(modelMatrix);
mat4.translate(modelMatrix, modelMatrix, translate);
mat4.scale(modelMatrix, modelMatrix, scale);

mat4.lookAt(
    viewMatrix,
    this.lightPos,
    this.focalPoint,
    this.lightUp
);

mat4.ortho(
    projectionMatrix,
    -100, 100,
    -100, 100,
    1, 200
);

mat4.multiply(lightMVP, projectionMatrix, viewMatrix);
mat4.multiply(lightMVP, lightMVP, modelMatrix);
```

含义：

- `modelMatrix`：模型局部空间到世界空间。
- `viewMatrix`：世界空间到光源观察空间。
- `projectionMatrix`：光源正交投影。
- `lightMVP = P * V * M`：模型顶点变换到光源裁剪空间。

初始正交投影范围计划使用：

```text
left = -100
right = 100
bottom = -100
top = 100
near = 1
far = 200
```

原因是当前场景中 Mary、地板和第二个 Mary 大致都落在这个范围内。后续如果阴影被裁剪，再调整范围。

### 2. 确认 shadow pass 生成深度图

现有 `WebGLRenderer.render()` 中已经有 shadow pass：

```js
if (this.lights[l].entity.hasShadowMap == true) {
    for (let i = 0; i < this.shadowMeshes.length; i++) {
        this.shadowMeshes[i].draw(this.camera);
    }
}
```

虽然这里传入的是主相机，但 shadow shader 的顶点位置实际使用：

```glsl
gl_Position = uLightMVP * vec4(aVertexPosition, 1.0);
```

因此第一版可以先不动渲染器，重点保证 `uLightMVP` 正确。

如果实现后阴影不对，再考虑在 shadow pass 前明确清理 FBO：

```js
gl.bindFramebuffer(gl.FRAMEBUFFER, light.fbo);
gl.clear(gl.COLOR_BUFFER_BIT | gl.DEPTH_BUFFER_BIT);
```

### 3. 实现 `useShadowMap()`

当前函数：

```glsl
float useShadowMap(sampler2D shadowMap, vec4 shadowCoord){
  return 1.0;
}
```

计划实现为：

```glsl
float useShadowMap(sampler2D shadowMap, vec4 shadowCoord) {
  vec3 projCoords = shadowCoord.xyz;

  if (projCoords.x < 0.0 || projCoords.x > 1.0 ||
      projCoords.y < 0.0 || projCoords.y > 1.0 ||
      projCoords.z < 0.0 || projCoords.z > 1.0) {
    return 1.0;
  }

  float closestDepth = unpack(texture2D(shadowMap, projCoords.xy));
  float currentDepth = projCoords.z;
  float bias = 0.005;

  return currentDepth - bias > closestDepth ? 0.0 : 1.0;
}
```

逻辑：

- 如果投影坐标超出 shadow map 范围，默认可见。
- 从 shadow map 取出光源视角下最近表面的深度。
- 与当前片元在 light space 的深度比较。
- 当前片元更远，说明被遮挡，返回 `0.0`。
- 否则返回 `1.0`。
- `bias` 用来缓解 shadow acne。

### 4. 任务 1 可能问题和调试方向

全黑：

- 可能是深度比较方向反了。
- 可能是 bias 太小。
- 可能是 light projection 范围不合适。
- 可能是 shadow map 没有正确写入。

完全没有阴影：

- `useShadowMap()` 始终返回 1。
- shadowCoord 超出 `[0, 1]` 后被默认可见。
- 正交投影范围过大导致深度精度差。
- `lightMVP` 矩阵顺序错误。

阴影位置偏移：

- `modelMatrix` 与正常渲染 pass 不一致。
- `vPositionFromLight` 使用的坐标和 shadow pass 不一致。

阴影锯齿：

- 硬阴影阶段允许存在。
- 后续 PCF 处理。

shadow acne：

- 先用常数 bias。
- 后续可改成基于法线和光线夹角的 bias。

例如：

```glsl
float bias = max(0.005 * (1.0 - dot(normalize(vNormal), normalize(uLightPos))), 0.001);
```
